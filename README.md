# FUTURE_CS_03 - API Security Risk Analysis
## Task 03

## Overview:
This project presents a security assessment of a public API (JSONPlaceholder) using safe, read-only testing methods.

## API Tested:
JSONPlaceholder (Test API)  
https://jsonplaceholder.typicode.com

## Tools Used:
- Postman (API request testing)
- Insomnia (API request testing and validation)
- Browser Developer Tools (Network traffic, headers, responses)

## Scope:
### Included:
- Read-only API testing (GET requests)
- Endpoint analysis
- Header inspection
- Response data analysis
- Authentication & authorization review

### Excluded: 
- Exploitation attempts
- Bypassing authentication
- Denial-of-Service (DoS) testing
- Testing private or production systems

## Methodology
The methodology followed a structured approach to API security analysis. The process began with reviewing the API documentation to understand its functionality and available endpoints. Relevant endpoints were then selected and tested using Postman and Insomnia. During testing, requests, headers, and responses were inspected using Browser Developer Tools to gain deeper insight into how the API behaves. Based on these observations, potential security risks were identified and analyzed. Each risk was then classified according to its severity level (Low, Medium, or High), and appropriate remediation strategies were proposed to address the identified vulnerabilities.
